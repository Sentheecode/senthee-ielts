# IELTS Agent 学习系统设计

## 1. 目标与边界

为一名在职、当前约大学英语四级水平的中国学习者建立个人私有学习系统。六个月后参加 IELTS，暂按澳洲工作/迁移所需的 General Training 准备，目标总分 7，并单独追踪四科达到 7 的进度。

系统服务于学习，不替代学习。MVP 只包含任务执行、提交反馈、复盘、学习贡献看板和 Agent 教练；不做社交、排行榜、课程商城或复杂内容后台。

## 2. 学习模型

- 白天输入：3–20 分钟的听力、短阅读、词块复习，适合工作等待间隙。
- 晚上输出：20–60 分钟的口语录音、写作和精听订正。
- 周末校准：完整或半套模考、周复盘、下一周计划。
- 任务按预计耗时分类为 3、10、25、60 分钟，不采用无法适应工作波动的固定课表。
- 首周使用官方样题完成四科诊断。在诊断前不展示虚构分数，只展示“待测”。

## 3. 产品形态

首版为响应式 PWA，覆盖 iPhone、Android 和桌面浏览器，并支持添加到手机主屏幕。学习闭环稳定使用 3–4 周后，再评估用 Capacitor 封装 iOS/Android App。原生能力只在确有需要时加入。

界面和教学反馈以简体中文为主，题目、原文、答案和口语训练保持英文。中文反馈重点识别中式直译、冠词、时态、单复数、搭配、连读弱读等常见问题。

## 4. 页面与核心流程

### 4.1 首页 / 今日

- 第一屏展示学习贡献图、今日积分、连续学习天数、四科状态和当前最优先任务。
- 用户选择当前可用时长，系统返回一项最适合的任务，而不是长任务清单。
- 完成后直接进入反馈或订正，避免任务“打卡完成但没有纠错”。

### 4.2 练习

- 听力、阅读：记录材料引用、题号、答案、正确性、耗时和错因。
- 写作：保存题目、草稿、终稿、Agent 反馈和用户二次修改。
- 口语：录音上传、语音转写、按评分维度反馈，并允许再次录制。
- 词块：支持听辨、拼写、释义、搭配和造句，不只做中英互认。

### 4.3 复盘

- 错题按知识不足、定位失败、同义替换未识别、拼写、审题、时间不足等原因分类。
- Agent 每次只给 1–3 个最高优先级问题，并生成对应的下一步任务。
- 用户订正后才获得主要的“有效学习积分”。

### 4.4 进度与 Agent

- 进度页展示四科估分、题型正确率、耗时、个人词块掌握度和周趋势。
- Agent 提供每日任务编排、写作教练、口语考官、错题解释和周复盘五种明确入口。
- AI 估分必须显示依据与置信度，并注明不等同于官方成绩。

## 5. 学习贡献看板

采用类似 GitHub contribution graph 的日历热力图，但颜色按有效学习积分计算：

- 完成输入任务获得基础积分。
- 完成订正、重做、写作和口语输出获得更高权重。
- 单纯打开页面、播放但未完成或挂机不计分。
- 点击某天可查看“学习 diff”：做了什么、订正了什么、掌握了哪些词块、能力指标有何变化。
- 每周由 Agent 生成周报：投入、有效产出、进步证据、主要瓶颈和下周调整。

积分用于展示个人连续性，不与他人比较，也不直接换算 IELTS 分数。四科估分仅在诊断或模考后更新。

## 6. 内容与词库

内容分三层：

1. IELTS 官方免费 General Training 样题，用于初始诊断和题型熟悉。
2. 用户个人持有的 Cambridge IELTS General Training 等正版材料，用于正式练习和模考。系统保存材料索引、用户作答与分析。
3. Agent 原创专项题，用于围绕薄弱项增加练习量，始终标注“AI 原创”，不作为正式模考分数来源。

词库由 Open English WordNet 等许可清晰的基础数据、预置高频词块、以及从用户错题/写作/口语中提取的个人词块组成。每张卡保存词义、音标、发音、搭配、例句、来源、掌握状态和下次复习时间。

## 7. 技术架构

- 前端：Next.js + TypeScript 的响应式 PWA，部署到 Vercel。
- 数据与身份：Supabase Auth + Postgres。
- 文件：Supabase Storage 保存个人口语录音；使用私有 bucket 和限时签名链接。
- AI 服务：只通过服务端调用模型，浏览器不持有模型密钥。模型能力分为转写、反馈、生成和任务编排。
- 后台任务：周报、间隔复习和较长音频处理使用异步任务；MVP 可先用数据库任务表和定时触发实现。
- 可移植性：核心数据可导出为 JSON/CSV；题目引用与用户作答分离，便于更换材料和模型。

主要实体包括 User、Goal、StudyTask、Attempt、MaterialRef、ErrorTag、VocabularyItem、ReviewSchedule、MockExam、BandEstimate、DailyContribution 和 AgentFeedback。

## 8. 失败处理与隐私

- 上传或 AI 请求失败时保存本地草稿并允许重试，不丢失写作与录音元数据。
- AI 返回无效结构时不写入正式估分，显示“反馈生成失败”并保留原始提交。
- 同一提交使用幂等键，避免重试导致重复积分。
- 私人材料、作答和音频默认仅本人可见；数据库启用行级权限。
- 用户可删除录音、练习记录或整个账号，并可导出学习数据。

## 9. 验证策略

- 单元测试：积分规则、间隔复习、任务选择、估分更新和幂等逻辑。
- 集成测试：登录、提交练习、上传录音、生成反馈、完成订正和更新贡献图。
- 端到端测试：手机端完成一项白天任务、晚间口语任务和周末模考流程。
- 内容测试：官方样题答案不得被 Agent 改写；AI 原创题必须带来源标签。
- 人工校准：定期将 AI 写作/口语反馈与官方评分描述或真人教师反馈对照。

## 10. 六个月使用节奏

- 第 1 周：诊断四科，建立设备与学习习惯。
- 第 2–6 周：补基础与熟悉题型，建立个人错题和词块库。
- 第 7–16 周：按薄弱题型专项训练，每两周一次阶段模考。
- 第 17–22 周：整套限时训练，重点修复写作和口语瓶颈。
- 第 23–24 周：考试节奏、睡眠和最终模考，不再大量增加新材料。

## 11. 首批材料

- The Official Cambridge Guide to IELTS：用于学习方法和四科技能。
- Cambridge IELTS 20 General Training：用于首次诊断后的正式练习与阶段模考。
- IELTS 官方免费 General Training 样题：首周即可开始使用。

在首次诊断前不购买更多专项书。根据诊断结果决定是否增加 Cambridge IELTS 19 General Training 或写作/语法专项材料。

## 12. MVP 成功标准

- 用户连续四周每周至少五天产生有效学习记录。
- 能在 10 秒内找到符合当前可用时长的下一项任务。
- 每次写作或口语提交都能完成“提交—反馈—二次修改/重录”的闭环。
- 看板能准确解释每一天的积分来源，不出现重复积分。
- 四周后能基于真实数据指出最主要的三个薄弱项，并据此调整下一周任务。
